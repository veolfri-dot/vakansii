# Фикс экранирования MarkdownV2 — символ `|`

## 🔴 Что было сломано

Telegram-бот не мог отправить ни одну вакансию в канал с ошибкой:

```
Can't parse entities: character '|' is reserved and must be escaped with the preceding '\'
```

Ошибка повторялась на всех 10 вакансиях из тестового набора.

### Root cause

1. **В `_escape_markdown_v2` не экранировался сам символ `\`**  
   Если входной текст уже содержал `\|` (или любой другой бэкслэш), итеративный `replace` создавал `\\|` → Telegram видел `\\` как экранированный бэкслэш, а `|` оставался неэкранированным → ошибка.

2. **В шаблонах сообщений использовался литеральный `|` как разделитель без экранирования**  
   Например:
   ```python
   f"{location}  |  {level_emoji} {level}"
   ```
   В MarkdownV2 `|` — зарезервированный символ в **любом** месте текста (не только в таблицах), поэтому парсер Telegram падал.

3. **`_escape_url` не обрабатывал `|` в URL**  
   Если ссылка содержала `|`, она попадала в `[текст](url)` неэкранированной и ломала MarkdownV2.

## ✅ Как починено

### 1. `_escape_markdown_v2` — заменён на regex single-pass

**Было:**
```python
escape_chars = r'_*[]()~`>#+-=|{}.!'
for char in escape_chars:
    text = text.replace(char, f'\{char}')
```

**Стало:**
```python
escape_chars = r'\_*[]()~`>#+-=|{}.!'
return re.sub(f'([{re.escape(escape_chars)}])', r'\\\1', text)
```

**Почему именно так:**
- Добавлен `\` в список экранируемых символов — устраняет двойное/некорректное экранирование.
- `re.sub` работает в один проход (non-overlapping), тогда как цепочка `replace` могла обработать уже вставленные бэкслэши и создать `\\|`.
- Это тот же подход, который используется в официальной библиотеке `python-telegram-bot` (`telegram.helpers.escape_markdown`).

### 2. Все литеральные `|` в f-шаблонах экранированы

Изменены строки-разделители вида ` | ` → ` \| ` во всех методах форматирования:
- `_format_compact` (2 шт.)
- `_format_full` (2 шт.)
- `format_job_list` (1 шт.)
- `format_enhanced_job_card` (1 шт.)
- `format_recommendations` (1 шт.)

Аналогичные строки найдены и исправлены в `channel_bot.py` (3 шт. в inline-методах бота).

### 3. `_escape_url` дополнен экранированием `|`

**Было:**
```python
return url.replace(')', '%29').replace('\\', '%5C')
```

**Стало:**
```python
return url.replace(')', '%29').replace('\\', '%5C').replace('|', '%7C')
```

**Почему именно так:**
- Внутри URL MarkdownV2-ссылки `|` должен быть процент-закодирован (`%7C`), иначе парсер Telegram считает его зарезервированным символом разметки.
- Добавлен аналогичный метод `_escape_url` в `channel_bot.py` и заменено использование `_escape_markdown_v2(url)` на `_escape_url(url)` в inline-ссылках.
